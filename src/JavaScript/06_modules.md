# モジュール（30問）
1. `<script type="module">`で読み込まれたファイルが通常のスクリプトと異なる点（スコープや実行順など）を説明してください。
2. ES Modulesが常に暗黙のstrict modeになる理由と、その影響を2つ挙げてください。
3. モジュールスコープで宣言した変数がグローバルに漏れない仕組みを説明してください。
4. 名前付きエクスポートの基本構文（宣言時エクスポートと後置エクスポート）を示してください。
5. デフォルトエクスポートの基本構文を示し、名前付きエクスポートとの違いを説明してください。
6. 複数の名前付きエクスポートを1行でまとめて書く方法を示してください。
7. インポート時にエイリアスを付ける構文（`import { foo as bar } from ...`）を示してください。
8. デフォルトエクスポートにエイリアスを付けてインポートする方法を示してください。
9. モジュール全体を名前空間として読み込む構文（`import * as ns from ...`）と、その使いどころを説明してください。
10. 副作用のみを目的としたインポート（`import './polyfill.js'`など）がどんなときに使われるか説明してください。
11. `export { foo as bar }`のようにエイリアス付きでエクスポートする意図と使いどころを説明してください。
12. 再エクスポート（`export { foo } from './a.js'`や`export * from './a.js'`）の挙動と注意点を説明してください。
13. デフォルトエクスポートを再エクスポートする構文例（`export { default as Foo } from './Foo.js'`）を書いてください。
14. 動的インポート（`import()`）の返り値と、読み込みタイミングの制御方法を説明してください。
15. トップレベル`await`が使える状況と、モジュールの評価順に与える影響を説明してください。
16. モジュールの評価は一度きりでキャッシュされる仕組みを説明し、副作用を伴う初期化処理での注意点を述べてください。
17. エクスポートされた値が「ライブバインディング」になるとはどういう意味か、例を挙げて説明してください。
18. インポートした名前付きバインディングを再代入できない理由と、代わりに行うべき対応を説明してください。
19. デフォルトエクスポートが1モジュールに1つだけ許可される理由と、スタイルガイド上の賛否を述べてください。
20. 名前付きエクスポートとデフォルトエクスポートのどちらを使うか決める際の判断基準を2つ挙げてください。
21. バンドラがツリーシェイキングを行う際にES Modulesが好まれる理由を説明してください。
22. 相対パス指定の際に拡張子が必要とされる理由と、ブラウザでの解決規則を説明してください。
23. ベアモジュール仕様（例: `import _ from 'lodash'`）がブラウザで動かない理由と、解決する方法を挙げてください。
24. Node.jsでESMとCommonJSを混在させるときの代表的な注意点を2つ挙げてください。
25. 循環参照があるモジュール同士を読み込んだ場合に起こり得る挙動と、設計上の回避策を説明してください。
26. JSONやCSSなど、非JavaScript資産をモジュールとして扱うときに一般的に必要となる仕組み（ローダー/バンドラ）を説明してください。
27. インポートの順序が評価結果に影響するケースと、安定した初期化順序を確保する方法を説明してください。
28. デフォルトエクスポートをインポートするときに波括弧が不要な理由を説明してください。
29. `this`がモジュールのトップレベルで`undefined`になる理由を説明してください。
30. モジュール設計のベストプラクティス（公開面の最小化、依存の明確化など）を3つ挙げてください。
