# デバッグと品質（30問）
1. `console.log`/`console.info`/`console.warn`/`console.error`の違いと、使い分けの基準を説明してください。
2. `console.table`で配列やオブジェクトを整形表示する例を書いてください。
3. `console.dir`が`console.log`と異なる表示をする理由と、DOM要素をデバッグするときの違いを説明してください。
4. `console.group`/`console.groupEnd`を使ってログを階層化する例を書いてください。
5. `console.time`/`console.timeEnd`で処理時間を測定する基本的な使い方を示してください。
6. `console.assert`がどのように動作するか説明し、失敗時の出力例を示してください。
7. ブラウザ開発者ツールでのソースマップの役割を説明し、ビルド済みコードのデバッグにどう役立つか述べてください。
8. `debugger`ステートメントをコードに挿入すると何が起こるか説明してください。
9. 条件付きブレークポイントを設定する利点と、典型的な利用シナリオを挙げてください。
10. ウォッチ式（watch expression）やスコープペインを使って値を追跡する方法を説明してください。
11. ネットワークパネルでAPIリクエストを調査するときに確認すべき情報を3つ挙げてください。
12. パフォーマンスパネル（タイムライン）でレイアウトスラッシングを見つける方法を説明してください。
13. メモリリークを疑ったときにヒープスナップショットを取る手順と、確認する指標を説明してください。
14. strict modeを有効にすると禁止される代表的なコード（暗黙的グローバルなど）を2つ挙げてください。
15. strict modeで`this`が未定義になるケースと、その理由を説明してください。
16. ESLintの役割と、プロジェクトに導入する目的を説明してください。
17. ESLintルールを一時的に無効化するコメント構文（`// eslint-disable-next-line`など）を示し、乱用を避けるべき理由を述べてください。
18. Prettierの役割と、ESLintとの役割分担を説明してください。
19. 自動整形ツールを導入するメリットと、コードレビューに与える影響を説明してください。
20. コーディング規約（命名・ファイル構成・インデントなど）を統一することがバグ低減に寄与する理由を述べてください。
21. 単体テストを書くことがリファクタリング時にどのように役立つか説明してください。
22. テストが検出しやすいバグと、静的解析で検出しやすいバグの違いを説明してください。
23. ログ出力の粒度（DEBUG/INFO/WARN/ERRORなど）を設計する際の指針を挙げてください。
24. `console`に秘密情報を書き出さないようにする理由と、誤出力を防ぐための対策を説明してください。
25. CIでLintとテストを自動実行することで得られる品質面のメリットを説明してください。
26. デバッグビルドと本番ビルドで挙動を変える際に注意すべき点（フラグ管理やデッドコード除去など）を述べてください。
27. エラー発生時にスクリーンショットや環境情報を記録することの利点を説明してください。
28. 例外を握り潰さずにユーザー通知とログ記録の両方を行う基本パターンを示してください。
29. フロントエンドの型チェックを強化するためにJSDocやTypeScriptを導入するメリットを説明してください。
30. 品質を保つための個人チェックリストを3項目作成し、日常的に確認する方法を述べてください。
