# 非同期処理（30問）
1. JavaScriptのイベントループにおけるタスクキューとマイクロタスクキューの違いを説明してください。
2. `setTimeout(fn, 0)`を呼び出したとき、コールバックはいつ実行されるかをイベントループの流れとともに説明してください。
3. Promiseの3状態（pending/fulfilled/rejected）と、一度settleしたら状態が変わらない理由を説明してください。
4. Promiseコンストラクタのエグゼキュータ関数が同期的に実行されることの意味と注意点を説明してください。
5. `then`で値を返す場合とPromiseを返す場合、それぞれ後続の`then`にどのように伝播するか説明してください。
6. `catch`と`then`の第2引数でのエラーハンドリングの違いと、推奨される書き方を述べてください。
7. `finally`が実行されるタイミングと、`finally`内の戻り値/例外が後続に与える影響を説明してください。
8. Promiseチェーンでエラーが伝播する仕組みを、例を用いて説明してください。
9. `Promise.resolve`と`Promise.reject`の使いどころと、既存のPromiseを渡したときの挙動を説明してください。
10. `Promise.all`の結果の配列順序と、どのような条件で拒否されるか説明してください。
11. `Promise.allSettled`の戻り値の構造を示し、どんなときに便利か説明してください。
12. `Promise.race`が最初に解決/拒否した結果を返す仕組みと、典型的な用途を挙げてください。
13. `Promise.any`が全て拒否された場合に発生する`AggregateError`の内容を説明してください。
14. `async`関数が常にPromiseを返すことの意味と、`return`/`throw`がどのようなPromiseに変換されるか説明してください。
15. `await`で非Promise値を待機したときの挙動を説明してください。
16. `await`が実行されたとき、後続処理がマイクロタスクとしていつ再開されるか説明してください。
17. 複数の非同期処理を直列に`await`した場合と、`Promise.all`で並列化した場合の違いを示してください。
18. `for await...of`で非同期イテラブルを反復する例を書き、同期イテラブルとの違いを説明してください。
19. 非同期ジェネレーター関数（`async function*`）の基本的な使い方と、`yield`で返される値の扱いを説明してください。
20. タイマーをキャンセルする方法（`clearTimeout`/`clearInterval`）と、キャンセルが効かないタイミングを説明してください。
21. デバウンスとスロットルの違いを説明し、それぞれ典型的に使われるユースケースを挙げてください。
22. イベントループがCPUバウンドな処理でブロックされると非同期処理にどんな影響が出るか説明してください。
23. マイクロタスクをキューに追加する`queueMicrotask`の用途を説明し、`Promise.resolve().then(...)`との違いを述べてください。
24. 非同期処理中にリソースをクリーンアップするためのパターン（`try/finally`など）を示してください。
25. `Promise`チェーン内で条件に応じて早期に処理を打ち切る方法を説明してください。
26. ユーザー操作によるキャンセルをPromiseで表現する方法を、`AbortController`以外の手法も含めて説明してください。
27. 未処理のPromise拒否が発生した場合にデバッグするための方法を2つ挙げてください。
28. `Promise.race`でタイムアウト制御を行うコード例を書いてください。
29. `async`関数内で例外を握り潰さないためのベストプラクティスを3つ挙げてください。
30. 非同期処理のテストを書く際、フレームワークに対してどのように完了を通知するか（`done`コールバックやPromise/async関数）を説明してください。
